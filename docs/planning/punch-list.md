# Punch List

Tiny isolated fixes. ≤30 minutes each, no design discussion needed. One of three planning trackers — see `CONTRIBUTING.md` → "The planning trackers."

## Workflow

When you sit down to work from this file:

1. Pick an item (or let the user pick one).
2. Read the one-liner. If it needs context to act on, ask the user — don't guess.
3. Do the fix. One item = one PR (or one commit, if you're not using PRs).
4. Delete the item from this file, in the same PR — the commit is the record (prune on resolve). *Not using git? Move it to a Closed section at the bottom instead.*
5. Don't batch-fix five items at once unless they touch the same code.

When you notice something punch-list-shaped:

1. Add it as a one-liner at the bottom (next number in sequence).
2. Don't fix it in the same PR as unrelated work.
3. If it's growing beyond 30 minutes or developing design questions, promote it to a side task or phase scope.
